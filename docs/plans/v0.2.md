# v0.2 実装プラン — 検索モード（Spotlight風テキスト検索）

アーキテクチャ・制約の詳細は `overview.md` を参照。
実装パターンは `mvp.md` および `HintMode/HintModeController.swift` を参照。

---

## v0.2 の完了条件

1. `Cmd+Shift+/` で検索モードが起動し、画面下端に検索バーが表示される
2. テキストを入力すると、AXTitle / AXLabel / AXDescription が一致するUI要素に緑の枠がハイライト表示される
3. `Enter` または `Space` でハイライト中の先頭要素をクリックし、モードを終了する
4. `Backspace` でクエリを1文字削除して再フィルタする
5. `ESC` でキャンセルし、オーバーレイが非表示になる
6. マッチゼロ件の場合、検索バーのテキストが赤くなり、Enter/Space は無効になる

### 1-2. SearchElementInfo 構造体を定義（SearchModeController.swift 内）

`UIElementInfo` と同様に値型のみで構成し、Sendable に準拠させる:

```swift
struct SearchElementInfo: Sendable {
    let frame: CGRect          // NSWindow座標系（原点:左下）変換済み
    let title: String          // AXTitle（空の場合あり）
    let label: String          // AXLabel（空の場合あり）
    let description: String    // AXDescription（空の場合あり）
    let role: String           // AXRole
    let axElement: AXElement   // @unchecked Sendable ラッパー
```

### 1-3. SearchModeController.swift の骨格

`HintModeController` と同一の構造パターンで実装する:

```swift
private enum SearchModeState {
    case inactive
    case active(elements: [SearchElementInfo], query: String, matched: [SearchElementInfo])
}

final class SearchModeController: @unchecked Sendable {
    private let axManager = AXManager()
    private var state: SearchModeState = .inactive
    private var isActive: Bool = false    // CGEventTap スレッドから読まれる
    private var searchView: SearchView?
    private weak var overlayWindow: OverlayWindow?
    private weak var hotkeyManager: HotkeyManager?

    func activate(overlayWindow: OverlayWindow, hotkeyManager: HotkeyManager) {
        guard !isActive else { return }
        self.overlayWindow = overlayWindow
        self.hotkeyManager = hotkeyManager

        guard let focusedApp = NSWorkspace.shared.frontmostApplication else { return }
        let appElement = AXUIElementCreateApplication(focusedApp.processIdentifier)

        axManager.fetchSearchableElements(in: appElement) { [weak self] elements in
            self?.startSearchMode(elements: elements, overlayWindow: overlayWindow, hotkeyManager: hotkeyManager)
        }
    }

    func deactivate() {
        isActive = false
        state = .inactive
        searchView?.removeFromSuperview()
        searchView = nil
        hotkeyManager?.keyEventHandler = nil
        overlayWindow?.hide()
    }

    // startSearchMode・handleKey は Phase 2 で実装
}
```

## Phase 2: テキスト入力 + フィルタリング + ハイライト描画

**目標:** 文字を入力するとUI要素名で絞り込まれ、マッチした要素に緑の枠が表示される。

### 2-1. UIElementEnumerator に enumerateSearchableElements() を追加

```swift
// UIElementEnumerator.swift に追加
static func enumerateSearchableElements(root: AXUIElement) -> [AXUIElement] {
    var result: [AXUIElement] = []
    collectVisible(element: root, into: &result, depth: 0)
    return result
}

private static func collectVisible(
    element: AXUIElement,
    into result: inout [AXUIElement],
    depth: Int
) {
    guard depth < 20, result.count < 500 else { return }  // 上限 500 件

    // AXHidden == true の要素はスキップ
    var hiddenRef: CFTypeRef?
    if AXUIElementCopyAttributeValue(element, "AXHidden" as CFString, &hiddenRef) == .success,
       let hidden = hiddenRef as? Bool, hidden { return }

    // AXTitle / AXLabel / AXDescription のいずれかが非空なら収集
    let searchKeys = ["AXTitle", "AXLabel", "AXDescription"]
    let hasText = searchKeys.contains { key in
        var ref: CFTypeRef?
        guard AXUIElementCopyAttributeValue(element, key as CFString, &ref) == .success,
              let s = ref as? String else { return false }
        return !s.trimmingCharacters(in: .whitespaces).isEmpty
    }
    if hasText { result.append(element) }

    var childrenRef: CFTypeRef?
    guard AXUIElementCopyAttributeValue(element, "AXChildren" as CFString, &childrenRef) == .success,
          let children = childrenRef as? [AXUIElement] else { return }
    for child in children {
        collectVisible(element: child, into: &result, depth: depth + 1)
    }
}
```

### 2-2. AXManager に fetchSearchableElements() を追加

既存の `fetchFrame()` を `internal` に変更し（`private` から変更）、`SearchElementInfo` 構築で再利用する:

```swift
// AXManager.swift に追加
func fetchSearchableElements(
    in app: AXUIElement,
    completion: @escaping @Sendable ([SearchElementInfo]) -> Void
) {
    let wrapped = AXElement(ref: app)
    let screenHeight = NSScreen.main?.frame.height ?? 0

    DispatchQueue.global(qos: .userInitiated).async {
        let elements = UIElementEnumerator.enumerateSearchableElements(root: wrapped.ref)
        let infos = elements.compactMap { el -> SearchElementInfo? in
            guard let frame = self.fetchFrame(element: el, screenHeight: screenHeight) else { return nil }

            func str(_ key: String) -> String {
                var ref: CFTypeRef?
                guard AXUIElementCopyAttributeValue(el, key as CFString, &ref) == .success,
                      let s = ref as? String else { return "" }
                return s
            }

            return SearchElementInfo(
                frame: frame,
                title: str("AXTitle"),
                label: str("AXLabel"),
                description: str("AXDescription"),
                role: str("AXRole"),
                axElement: AXElement(ref: el)
            )
        }
        DispatchQueue.main.async { completion(infos) }
    }
}
```

### 2-3. SearchModeController.filter() 純粋関数

テスト容易性のため `static` 純粋関数として実装する:

```swift
// SearchModeController に追加
static func filter(
    elements: [SearchElementInfo],
    query: String
) -> [SearchElementInfo] {
    guard !query.isEmpty else { return elements }
    let q = query.lowercased()
    return elements.filter { $0.searchableText.contains(q) }
}
```

### 2-4. startSearchMode() と handleKey() の実装

```swift
private func startSearchMode(
    elements: [SearchElementInfo],
    overlayWindow: OverlayWindow,
    hotkeyManager: HotkeyManager
) {
    state = .active(elements: elements, query: "", matched: [])
    isActive = true

    let view = SearchView(frame: overlayWindow.contentView?.bounds ?? .zero)
    view.autoresizingMask = [.width, .height]
    view.update(query: "", matched: [])
    overlayWindow.contentView?.addSubview(view)
    searchView = view

    overlayWindow.show()

    hotkeyManager.keyEventHandler = { [weak self] keyCode, flags in
        guard let self, self.isActive else { return false }
        DispatchQueue.main.async { self.handleKey(keyCode: keyCode, flags: flags) }
        return true
    }
}

private func handleKey(keyCode: CGKeyCode, flags: CGEventFlags) {
    guard case .active(let elements, let query, _) = state else { return }

    // ESC: 常に有効
    if keyCode == 53 { deactivate(); return }

    // Backspace: クエリを1文字削除
    if keyCode == 51 {
        let newQuery = String(query.dropLast())
        let matched = SearchModeController.filter(elements: elements, query: newQuery)
        state = .active(elements: elements, query: newQuery, matched: matched)
        searchView?.update(query: newQuery, matched: matched)
        return
    }

    // Enter / Space: 先頭マッチをクリック
    if keyCode == 36 || keyCode == 49 {
        guard case .active(_, _, let matched) = state, let first = matched.first else { return }
        let element = first.axElement.ref
        deactivate()
        DispatchQueue.main.asyncAfter(deadline: .now() + 0.05) { [weak self] in
            self?.axManager.click(element: element)
        }
        return
    }

    // 修飾キー付きは無視
    guard flags.intersection([.maskCommand, .maskControl, .maskAlternate]).isEmpty else { return }
    guard let char = keyCodeToChar(keyCode) else { return }

    let newQuery = query + char
    let matched = SearchModeController.filter(elements: elements, query: newQuery)
    state = .active(elements: elements, query: newQuery, matched: matched)
    searchView?.update(query: newQuery, matched: matched)
}

// HintModeController の keyCodeToChar と同一内容
private func keyCodeToChar(_ keyCode: CGKeyCode) -> String? {
    let map: [CGKeyCode: String] = [
        0: "a", 11: "b", 8: "c", 2: "d", 14: "e",
        3: "f", 5: "g", 4: "h", 34: "i", 38: "j",
        40: "k", 37: "l", 46: "m", 45: "n", 31: "o",
        35: "p", 12: "q", 15: "r", 1: "s", 17: "t",
        32: "u", 9: "v", 13: "w", 7: "x", 16: "y", 6: "z",
        // 数字キー
        29: "0", 18: "1", 19: "2", 20: "3", 21: "4",
        23: "5", 22: "6", 26: "7", 28: "8", 25: "9",
    ]
    return map[keyCode]
}
```

注意: `keyCodeToChar` は `HintModeController` と重複する。将来的に共通ユーティリティ（`KeyCodeMapper.swift`）へ抽出することを推奨するが、v0.2 では各コントローラ内にコピーして実装し、重複解消は別タスクとする。

### 2-5. SearchView の draw() を実装

```swift
override func draw(_ dirtyRect: NSRect) {
    super.draw(dirtyRect)
    drawHighlights()
    drawSearchBar()
}

private func drawHighlights() {
    for info in matchedElements {
        let path = NSBezierPath(roundedRect: info.frame.insetBy(dx: -2, dy: -2), xRadius: 4, yRadius: 4)
        NSColor.systemGreen.withAlphaComponent(0.85).setStroke()
        path.lineWidth = 2.5
        path.stroke()

        NSColor.systemGreen.withAlphaComponent(0.12).setFill()
        path.fill()
    }
}

private func drawSearchBar() {
    let barHeight: CGFloat = 52
    let barRect = CGRect(x: 0, y: 0, width: bounds.width, height: barHeight)
    NSColor.black.withAlphaComponent(0.80).setFill()
    NSBezierPath(roundedRect: barRect, xRadius: 0, yRadius: 0).fill()

    let isNoMatch = !query.isEmpty && matchedElements.isEmpty
    let textColor: NSColor = isNoMatch ? .systemRed : (query.isEmpty ? .systemGray : .white)
    let displayText = query.isEmpty ? "検索... (Cmd+Shift+/)" : query

    let attrs: [NSAttributedString.Key: Any] = [
        .font: NSFont.monospacedSystemFont(ofSize: 15, weight: .regular),
        .foregroundColor: textColor
    ]
    (displayText as NSString).draw(at: CGPoint(x: 20, y: 17), withAttributes: attrs)

    if !query.isEmpty {
        let countText = "\(matchedElements.count) 件"
        let countAttrs: [NSAttributedString.Key: Any] = [
            .font: NSFont.systemFont(ofSize: 12, weight: .regular),
            .foregroundColor: NSColor.systemGray
        ]
        let countSize = (countText as NSString).size(withAttributes: countAttrs)
        (countText as NSString).draw(
            at: CGPoint(x: bounds.width - countSize.width - 20, y: 19),
            withAttributes: countAttrs
        )
    }
}
```

## 技術的注意点

### Swift 6 対応（CRITICAL）

- `kAXTitleAttribute` 等のCグローバルは **使わない**。生の文字列 `"AXTitle"` 等を使う
- `AXUIElement` は `Sendable` 非準拠。バックグラウンド→メイン間の受け渡しは `AXElement`（`@unchecked Sendable`）でラップする
- `SearchModeController` は `@unchecked Sendable` で宣言。`isActive` フラグのみ CGEventTap スレッドから読まれ、他の状態操作はすべてメインスレッドで行う

### スレッド安全性

- `fetchSearchableElements()` はバックグラウンドキューで列挙し、completion はメインスレッドで呼ぶ
- `state` の読み書きはすべてメインスレッドで行う
- `overlayWindow.show()` / `.hide()` はメインスレッドで呼ぶ

### Enter/Space とホットキーの競合

`keyEventHandler` がセットされている間はホットキー判定に到達しないため競合は発生しない（`HotkeyManager.handleEvent` の委譲パターン）。

### AXLabel vs AXDescription

- macOS アプリによって `AXLabel` を使うものと `AXDescription` を使うものが混在する
- 両方を `searchableText` に含めることで網羅性を高める
- Electron アプリは AX ツリーが限定的なため部分対応（既知の制約）

### 要素数の上限

`enumerateSearchableElements()` 内で `result.count < 500` のガードを設ける。

## Bug Fix: Enter/Space でクリックされない問題

### Bug 1: 空クエリ時の matched が空

`startSearchMode` の初期化で `matched: []` としているため、何も入力せずに Enter/Space を押しても `matched.first` が `nil` でガードが通らない。

**修正:** 初期状態でも全要素を matched に入れる。

```swift
// Before
state = .active(elements: elements, query: "", matched: [])

// After
state = .active(elements: elements, query: "", matched: elements)
```

合わせて `SearchView.update()` の初期呼び出しも修正:

```swift
view.update(query: "", matched: elements)
```

### Bug 2: AXPress が効かない要素がある（主因）

`enumerateSearchableElements()` はテキスト属性を持つ全要素（ラベル・ヘッダー等の非インタラクティブ要素を含む）を収集するが、`AXUIElementPerformAction(element, "AXPress")` はボタン・リンク等のインタラクティブ要素にしか効かず、マッチ先頭がラベルだと静かに失敗する。

**修正:** `AXPress` の代わりに CGEvent マウスクリックを使う。要素の中心座標にクリックイベントを送るため、AX サポートに依存しない。

`AXManager` に `clickAt(frame:)` を追加する:

```swift
// AXManager.swift に追加
func clickAt(frame: CGRect) {
    // frame は NSWindow 座標系（原点:左下）なのでスクリーン座標（原点:左上）に戻す
    let screenHeight = NSScreen.main?.frame.height ?? 0
    let centerX = frame.origin.x + frame.width / 2
    let centerY = screenHeight - frame.origin.y - frame.height / 2
    let point = CGPoint(x: centerX, y: centerY)

    let down = CGEvent(mouseEventSource: nil, mouseType: .leftMouseDown,
                       mouseCursorPosition: point, mouseButton: .left)
    let up   = CGEvent(mouseEventSource: nil, mouseType: .leftMouseUp,
                       mouseCursorPosition: point, mouseButton: .left)
    down?.post(tap: .cghidEventTap)
    up?.post(tap: .cghidEventTap)
}
```

`SearchModeController.handleKey()` のクリック処理を修正:

```swift
// Before
deactivate()
DispatchQueue.main.asyncAfter(deadline: .now() + 0.05) { [weak self] in
    self?.axManager.click(element: element)
}

// After
let frame = first.frame
deactivate()
DispatchQueue.main.asyncAfter(deadline: .now() + 0.05) { [weak self] in
    self?.axManager.clickAt(frame: frame)
}
```
