## v0.3 の完了条件

1. `Cmd+Shift+J` でスクロールモードが起動し、画面左下に "SCROLL" バナーが表示される
2. `j` で下スクロール（約 60px）、`k` で上スクロール（約 60px）
3. `d` で半ページ下スクロール（約 300px）、`u` で半ページ上スクロール（約 300px）
4. `ESC` でスクロールモードを終了し、バナーが消える
5. HintMode / SearchMode がアクティブな間は Cmd+Shift+J を無視する

### 1-3. ScrollModeController.swift の骨格

`HintModeController` と同一の構造パターンで実装する:

```swift
private enum ScrollModeState {
    case inactive
    case active
}

final class ScrollModeController: @unchecked Sendable {
    private var state: ScrollModeState = .inactive
    private var isActive: Bool = false    // CGEventTap スレッドから読まれる
    private var indicatorView: ScrollIndicatorView?
    private weak var overlayWindow: OverlayWindow?
    private weak var hotkeyManager: HotkeyManager?

    func activate(overlayWindow: OverlayWindow, hotkeyManager: HotkeyManager) {
        guard !isActive else { return }
        self.overlayWindow = overlayWindow
        self.hotkeyManager = hotkeyManager
        state = .active
        isActive = true

        let indicator = ScrollIndicatorView(frame: CGRect(x: 12, y: 12, width: 80, height: 28))
        overlayWindow.contentView?.addSubview(indicator)
        self.indicatorView = indicator
        overlayWindow.show()

        hotkeyManager.keyEventHandler = { [weak self] keyCode, flags in
            guard let self, self.isActive else { return false }
            DispatchQueue.main.async { self.handleKey(keyCode: keyCode, flags: flags) }
            return true
        }
    }

    func deactivate() {
        isActive = false
        state = .inactive
        currentTargetPoint = nil
        indicatorView?.removeFromSuperview()
        indicatorView = nil
        hotkeyManager?.keyEventHandler = nil
        overlayWindow?.hide()
    }

    private func handleKey(keyCode: CGKeyCode, flags: CGEventFlags) {
        // Phase 2 で実装
    }
}
```

## Phase 2: キー入力 → CGEvent スクロール送信

**目標:** `j/k/u/d` で実際のスクロールが発生する。

### スクロール量の設計

| キー | keyCode | 方向 | wheel1 値 | px 目安 |
|-----|---------|------|-----------|--------|
| `j` | 38 | 下 | -3 | 約 60px |
| `k` | 40 | 上 | +3 | 約 60px |
| `d` | 2 | 下 | -15 | 約 300px |
| `u` | 32 | 上 | +15 | 約 300px |

`wheel1` 正 = 上スクロール、負 = 下スクロール（macOS 標準）。

### 2-1. ScrollEngine.swift の実装

```swift
import CoreGraphics

enum ScrollDirection {
    case up, down
}

struct ScrollAmount {
    let lines: Int32   // CGEvent wheel1 に渡す（正=上、負=下）

    static func step(direction: ScrollDirection) -> ScrollAmount {
        ScrollAmount(lines: direction == .up ? 3 : -3)
    }

    static func halfPage(direction: ScrollDirection) -> ScrollAmount {
        ScrollAmount(lines: direction == .up ? 15 : -15)
    }
}

enum ScrollEngine {
    static func scroll(
        amount: ScrollAmount,
        targetPoint: CGPoint? = nil,
        source: CGEventSource? = nil
    ) {
        let event = CGEvent(
            scrollWheelEvent2Source: source,
            units: .line,
            wheelCount: 1,
            wheel1: amount.lines,
            wheel2: 0,
            wheel3: 0
        )
        if let point = targetPoint {
            event?.location = point
        }
        event?.post(tap: .cghidEventTap)
    }
}
```

`units: .line` を使う理由: `.pixel` はキーリピート時に過剰スクロールになるため。

### 2-2. キーコード定数の定義

`ScrollModeController.swift` のファイルトップで `private enum` として定義し、マジックナンバーを禁止する:

### 2-3. handleKey() の実装

```swift
private func handleKey(keyCode: CGKeyCode, flags: CGEventFlags) {
    // ESC は常に有効
    if keyCode == ScrollKeyCode.esc {
        deactivate()
        return
    }

    // 修飾キー付きは無視（Cmd+J 等のシステムショートカットとの競合防止）
    guard flags.intersection([.maskCommand, .maskControl, .maskAlternate]).isEmpty else { return }

    switch keyCode {
    case ScrollKeyCode.j:
        ScrollEngine.scroll(amount: .step(direction: .down), targetPoint: currentTargetPoint)
    case ScrollKeyCode.k:
        ScrollEngine.scroll(amount: .step(direction: .up), targetPoint: currentTargetPoint)
    case ScrollKeyCode.d:
        ScrollEngine.scroll(amount: .halfPage(direction: .down), targetPoint: currentTargetPoint)
    case ScrollKeyCode.u:
        ScrollEngine.scroll(amount: .halfPage(direction: .up), targetPoint: currentTargetPoint)
    default:
        break
    }
}
```

## Phase 3: スクロール対象の精度向上

**目標:** AXUIElement でスクロール可能領域を特定し、正しい領域にスクロールイベントを送信する。

Phase 2 では `CGEvent.post(tap: .cghidEventTap)` でシステム全体にイベントを送る。これはカーソルが乗っている要素にスクロールが届くため多くのケースで機能するが、スクロールモード起動後にカーソルが別の要素に移動した場合に意図しない領域がスクロールされる問題がある。Phase 3 でこれを解決する。

### 3-1. ScrollTarget.swift の実装

```swift
import AppKit

enum ScrollTarget {
    /// フォーカスアプリの AXFocusedUIElement からスクロール可能な祖先を探す
    /// 見つからなければ nil を返す（呼び出し元はターゲットなしスクロールにフォールバック）
    static func findScrollableElement(in app: AXUIElement) -> AXUIElement? {
        var focusedRef: CFTypeRef?
        guard AXUIElementCopyAttributeValue(app, "AXFocusedUIElement" as CFString, &focusedRef) == .success,
              let focusedRef else { return nil }
        let focused = focusedRef as! AXUIElement

        if isScrollable(element: focused) { return focused }
        return findScrollableAncestor(of: focused, depth: 0)
    }

    static func isScrollable(element: AXUIElement) -> Bool {
        var roleRef: CFTypeRef?
        guard AXUIElementCopyAttributeValue(element, "AXRole" as CFString, &roleRef) == .success,
              let role = roleRef as? String else { return false }
        return scrollableRoles.contains(role)
    }

    private static let scrollableRoles: Set<String> = [
        "AXScrollArea", "AXWebArea", "AXTextArea", "AXList", "AXTable", "AXOutline"
    ]

    private static func findScrollableAncestor(of element: AXUIElement, depth: Int) -> AXUIElement? {
        guard depth < 10 else { return nil }
        var parentRef: CFTypeRef?
        guard AXUIElementCopyAttributeValue(element, "AXParent" as CFString, &parentRef) == .success,
              let parentRef else { return nil }
        let parent = parentRef as! AXUIElement
        if isScrollable(element: parent) { return parent }
        return findScrollableAncestor(of: parent, depth: depth + 1)
    }

    /// AXUIElement の中心点をスクリーン座標系（原点:左上）で返す
    /// CGEvent.location にそのまま使える（NSWindow座標変換は不要）
    static func centerPoint(of element: AXUIElement) -> CGPoint? {
        var posRef: CFTypeRef?
        var sizeRef: CFTypeRef?
        guard AXUIElementCopyAttributeValue(element, "AXPosition" as CFString, &posRef) == .success,
              AXUIElementCopyAttributeValue(element, "AXSize" as CFString, &sizeRef) == .success,
              let posRef, let sizeRef else { return nil }

        var position = CGPoint.zero
        var size = CGSize.zero
        AXValueGetValue(posRef as! AXValue, .cgPoint, &position)
        AXValueGetValue(sizeRef as! AXValue, .cgSize, &size)

        guard size.width > 0, size.height > 0 else { return nil }
        return CGPoint(x: position.x + size.width / 2, y: position.y + size.height / 2)
    }
}
```

### 3-2. ScrollModeController に currentTargetPoint を追加

```swift
// ScrollModeController に追加
private var currentTargetPoint: CGPoint? = nil  // メインスレッドでのみ読み書き
```

`activate()` 時にバックグラウンドでスクロール対象を探索し、`currentTargetPoint` に保持する:

```swift
func activate(overlayWindow: OverlayWindow, hotkeyManager: HotkeyManager) {
    guard !isActive else { return }
    // ... （バナー表示・keyEventHandler セット）...

    // バックグラウンドでスクロール対象の中心点を取得
    guard let focusedApp = NSWorkspace.shared.frontmostApplication else { return }
    let appElement = AXElement(ref: AXUIElementCreateApplication(focusedApp.processIdentifier))
    DispatchQueue.global(qos: .userInitiated).async { [weak self] in
        let target = ScrollTarget.findScrollableElement(in: appElement.ref)
        let point = target.flatMap { ScrollTarget.centerPoint(of: $0) }
        DispatchQueue.main.async { self?.currentTargetPoint = point }
    }
}
```

`deactivate()` でリセット（Phase 1 の骨格に含め済み）:

```swift
func deactivate() {
    // ...
    currentTargetPoint = nil
    // ...
}
```

## テストファイル: Tests/VimursorTests/ScrollEngineTests.swift

```swift
import XCTest
@testable import Vimursor

final class ScrollEngineTests: XCTestCase {

    func testStepDownIsNegative() {
        let amount = ScrollAmount.step(direction: .down)
        XCTAssertLessThan(amount.lines, 0, "下スクロールは負の wheel1 値")
    }

    func testStepUpIsPositive() {
        let amount = ScrollAmount.step(direction: .up)
        XCTAssertGreaterThan(amount.lines, 0, "上スクロールは正の wheel1 値")
    }

    func testHalfPageDownHasLargerMagnitudeThanStep() {
        let step = ScrollAmount.step(direction: .down)
        let halfPage = ScrollAmount.halfPage(direction: .down)
        XCTAssertLessThan(halfPage.lines, step.lines, "半ページは1ステップより大きな絶対値を持つ")
    }

    func testHalfPageUpHasLargerMagnitudeThanStep() {
        let step = ScrollAmount.step(direction: .up)
        let halfPage = ScrollAmount.halfPage(direction: .up)
        XCTAssertGreaterThan(halfPage.lines, step.lines)
    }

    func testUpDownSymmetry() {
        let down = ScrollAmount.step(direction: .down)
        let up = ScrollAmount.step(direction: .up)
        XCTAssertEqual(down.lines, -up.lines, "上下は符号反転で対称")
    }

    func testHalfPageUpDownSymmetry() {
        let down = ScrollAmount.halfPage(direction: .down)
        let up = ScrollAmount.halfPage(direction: .up)
        XCTAssertEqual(down.lines, -up.lines)
    }

    func testStepAndHalfPageRatio() {
        let step = abs(Int(ScrollAmount.step(direction: .down).lines))
        let halfPage = abs(Int(ScrollAmount.halfPage(direction: .down).lines))
        XCTAssertGreaterThan(halfPage, step * 3, "半ページは1ステップの3倍以上")
    }
}
```

## 技術的注意点

### CGEvent.scrollWheelEvent2 の挙動（CRITICAL）

- `units: .line` を使う（`.pixel` はキーリピート時に過剰スクロールになる）
- `wheel1` 正 = 上スクロール、負 = 下スクロール（直感と**逆**。要注意）
- `wheelCount: 1` で十分（横スクロール `wheel2` は不要）
- `post(tap: .cghidEventTap)` でシステムに注入

### Swift 6 対応（CRITICAL）

- `kAXScrollAreaRole` 等のCグローバルは使わない。生の文字列 `"AXScrollArea"` 等を使う
- `ScrollModeController` は `@unchecked Sendable` で宣言
- `ScrollEngine` は副作用のある `static func` を持つが、状態を持たないため concurrency-safe
- `currentTargetPoint`（`CGPoint?`）はメインスレッドでのみ読み書きする
- `AXUIElement` の受け渡しは `AXElement`（`@unchecked Sendable`）でラップする

### AX座標と CGEvent.location の座標系（重要）

- `AXUIElement` の `AXPosition` はスクリーン座標系（原点:左上）
- `CGEvent.location` もスクリーン座標系（原点:左上）
- **変換は不要**（NSWindow座標系への変換は HintView の描画専用）

### j キーコード（38）とホットキーの競合

- 起動ホットキー `Cmd+Shift+J`（keyCode: 38、flags: cmdShift）
- スクロール `j`（keyCode: 38、flags: なし）
- `keyEventHandler` がセットされている間はホットキー判定に到達しないため競合しない

### Electron アプリへの対応

VS Code 等の Electron アプリは AXUIElement ツリーが限定的なため、Phase 3 の `ScrollTarget` が正しいスクロール領域を見つけられない場合がある。その場合は `targetPoint: nil`（カーソル位置スクロール）にフォールバックされる。既知の制約として `overview.md` に記録すること。
